    计网06 网络模型

    佳成 2022/4/6

    图解

    # 1. OSI七层模型

    详细图解

    # 1）应用层

    作用

    ​ 为应用程序提供服务。

    协议

    • HTTP（超文本传输协议）
    • FTP（文件传输协议）
    • TELNET（远程登录协议）
    • SMTP（电子邮件协议）

    # 2）表示层

    作用

    ​ 主要负责数据格式的转换，数据的压缩、加密、打包等功能也在这层完成。

    协议

    • ASCII ：用于格式转换
    • SSL/TLS: 用于加密通信

    # 3）会话层

    作用

    ​ 负责建立和断开通信连接，以及数据的分割等数据传输相关的管理。

    协议

    • ADSP：AppleTalk数据流协议
    • RPC：远程过程调用协议

    # 4）传输层

    作用

    ​ 负责可靠传输、指定通信端口

    协议

    • TCP: 传输控制协议
    • UDP：用户数据报协议

    # 5）网络层

    作用

    ​ 网络层负责将数据传输到目标地址。目标地址可以使多个网络通过路由器连接而成的某个地址， 所以这层主要负责寻址和路由选择。

    协议

    • IP：网际互连协议
    • ICMP：Internet控制报文协议

    # 6）数据链路层

    作用

    ​ 负责互连设备之间传送和识别数据帧。

    协议

    • HDLC
    • PPP
    • SLIP

    # 7）物理层

    作用

    ​ 将数据的0、1转换成电信号和光信号传输给物理的传输介质。

    协议

    • RS232C
    • RS 449/422/423
    • V.24
    • X.21
    • X.21bis

    # 2. 五层模型

    # 1）应用层

    作用

    直接为应用进程提供服务。

    协议

    • HTTP协议： 超文本传输协议
    • FTP协议： 文件传输
    • SMTP协议： 电子邮件
    • DNS： 域名查询

    ​

    # 2）传输层

    作用

    负责为两台主机中的进程提供通信服务，指定端口。

    协议

    • TCP ： 传输层控制协议（Transmission Control Protocol），大部分应用使用的正是 TCP 传输层协议，比如 HTTP 应用层协议。TCP 相比 UDP 多了很多特性，比如流量控制、超时重传、拥塞控制等，这些都是为了保证数据包能可靠地传输给对方。

    • UDP : 只负责发送数据包，不保证数据包是否能抵达对方，但它实时性相对更好，传输效率也高。

    # 3） 网络层

    作用

    ​ 负责路径和节点进行选择。

    协议

    • IP协议，IP 协议会将传输层的报文作为数据部分，再加上 IP 包头组装成 IP 报文，如果 IP 报文大小超过 MTU（以太网中一般为 1500 字节）就会再次进行分片，得到一个即将发送到网络的 IP 报文。

    # 4）数据链路层

    作用

    ​ 负责将网络层交下来的 IP 数据报封装成帧，并在链路的两个相邻节点间传送帧，每一帧都包含数据和必要的控制信息（如同步信息、地址信息、差错控制等）。

    协议

    • ARP协议： 转换IP为MAC

    # 5）物理层

    作用

    ​ 当数据准备从设备发送到网络时，需要将数据包转换成电信号，让其可以在物理介质中传输，这一层就是物理层，主要为数据链路层提供二进制传输的服务。

    # 3. TCP/IP参考模型（四层）

    # 1）应用层

    作用

    提供应用程序服务

    # 2）传输层

    作用

    为两台主机设备提供端到端的通信

    # 3）网际层

    作用

    ​ 负责处理IP数据包的传输、路由、建立主机间的通信

    # 4）网络接口层

    作用

    ​ 包含了OSI模型的物理层和数据链路层
